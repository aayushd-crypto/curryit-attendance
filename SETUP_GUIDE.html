🍛 CURRYiT — Complete Setup Guide

Step-by-step instructions to deploy your Attendance Management System. No coding knowledge required.

1
Create Your Supabase Database (5 minutes)
1.1 — Create a free Supabase account

Go to supabase.com and click "Start your project". Sign up with your Google account or email.

1.2 — Create a new project

Click "New project". Fill in:

Name: curryit-attendance
Database password: Create a strong password and save it somewhere safe
Region: South Asia (Mumbai) — closest to Delhi

Wait 2 minutes for the project to spin up.

1.3 — Run the database schema

In the Supabase dashboard, click SQL Editor in the left sidebar.
Click "New query".
Open the file supabase/migrations/001_complete_schema.sql from the project folder.
Copy the entire contents and paste it into the SQL Editor.
Click the green "Run" button.

✓ You should see "Success. No rows returned." — this means all tables were created.
1.4 — Save your API keys

In the Supabase dashboard, go to Project Settings → API.

Copy these two values — you'll need them shortly:

Project URL: looks like https://abcdefgh.supabase.co

anon public key: a very long string starting with "eyJ..."
2
Create Your GitHub Account & Upload the Code (10 minutes)
2.1 — Create a GitHub account

Go to github.com and sign up for a free account. Think of GitHub as a cloud storage specifically for code.

2.2 — Create a new repository

After logging in, click the "+" icon in the top right → "New repository".
Set name to curryit-attendance, make it Private, then click "Create repository".

2.3 — Upload the project files

On your new repository page, click "uploading an existing file" link.
Drag the entire curryit-attendance folder contents into the upload area.
Important: Do NOT upload the node_modules folder if it exists. Only upload the code files.
Click "Commit changes".

⚠️ Do NOT upload the .env.local file — it contains secret keys. The .gitignore file prevents this automatically.
3
Deploy on Vercel (5 minutes)
3.1 — Create a Vercel account

Go to vercel.com and click "Sign Up". Choose "Continue with GitHub" — this connects your GitHub account automatically.

3.2 — Import your project

On the Vercel dashboard, click "Add New Project".
You'll see your curryit-attendance repository. Click "Import".

3.3 — Add environment variables

Before clicking Deploy, scroll down to "Environment Variables". Add these two:

Name: VITE_SUPABASE_URL Value: (paste your Supabase Project URL here) Name: VITE_SUPABASE_ANON_KEY Value: (paste your Supabase anon key here)
3.4 — Deploy!

Click the big "Deploy" button. Vercel will build and deploy your app in about 2 minutes.

✓ You'll get a live URL like: curryit-attendance.vercel.app
4
Create User Accounts (10 minutes)
4.1 — Create your Super Admin account

In Supabase, go to Authentication → Users → Invite user.
Enter your email. After the user is created, go to SQL Editor and run:

-- Replace the email with your actual email UPDATE profiles SET role = 'super_admin', full_name = 'Your Name Here' WHERE email = 'your-email@curryit.in';
4.2 — Create accounts for other users

For each employee, admin, or coordinator, go to Authentication → Users → Invite user.
After creating the user, update their role and link to employee record:

-- For an Admin: UPDATE profiles SET role = 'admin', full_name = 'Admin Name' WHERE email = 'admin@curryit.in'; -- For CMK Coordinator: UPDATE profiles SET role = 'cmk_coordinator', full_name = 'Coordinator Name' WHERE email = 'coordinator@curryit.in'; -- For Employee (link to their employee record): UPDATE profiles SET role = 'employee', full_name = 'Employee Name', employee_id = (SELECT id FROM employees WHERE email = 'emp@curryit.in') WHERE email = 'emp@curryit.in';
4.3 — Add employees via the app

Log in as Super Admin → Go to Employees → Click "Add Employee".
Fill in all the details. The employee's email must match their login email.

User Roles & Permissions Reference
Role What they can do What they cannot do
Super Admin Everything — manage users, all employees, all reports, all settings Nothing restricted
Admin View all attendance, approve leaves, view reports, manage employees Cannot change system settings or manage admins
CMK Coordinator Mark CMK employees' attendance, view CMK reports Cannot see Office employees, cannot approve leave, cannot access settings
Employee Mark own attendance (Office/Remote), apply leave, view own history Cannot see other employees' data
6
Daily Operations Guide

For Office Employees (daily)

1. Open the app on phone or computer
2. Log in with your email and password
3. Click "Mark Present"
4. Select In Office or Remote
5. Done! ✓

For CMK Coordinator (daily)

1. Log in → Go to "CMK Attendance"
2. For each employee, click the circle under P (Present), A (Absent), or L (Leave)
3. Use "Mark all present" button for bulk marking, then adjust exceptions
4. Click "Save Attendance"
5. Done! ✓

For Admin (weekly)

1. Check the Dashboard for today's summary
2. Go to Leave to approve/reject any pending requests
3. Go to Reports to download monthly attendance Excel/PDF

Troubleshooting & Support
App shows blank screen or error

Check that your VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY are correctly set in Vercel environment variables. In Vercel, go to Project → Settings → Environment Variables.

Employee can't log in

Make sure you: (1) created the user in Supabase Auth, (2) linked the profile to an employee record, (3) the employee record email matches the auth email exactly.

Attendance already marked error

The system blocks duplicate attendance for the same day. This is intentional — one entry per person per day.
